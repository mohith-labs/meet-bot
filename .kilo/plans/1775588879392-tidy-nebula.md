# Fix: Stop Bot Hangs Indefinitely ("Stopping" forever)

## Root Cause

The `stopBot()` method in `backend/src/bots/google-meet-bot.service.ts` hangs because of **two blocking calls**:

### 1. Primary: `video.saveAs()` deadlock (line ~1721)

```typescript
// CURRENT CODE (BROKEN):
// Save screen recording before closing context
if (bot.screenRecordingEnabled && bot.storagePath) {
  const video = bot.page.video();
  if (video) {
    await video.saveAs(screenPath);  // ← HANGS FOREVER
  }
}
// ... later ...
await bot.context.close();  // ← Never reached
```

Playwright's `video.saveAs()` waits for the video file to be finalized, which only happens when the page or browser context is closed. Since we call `saveAs()` BEFORE `context.close()`, it creates a deadlock: `saveAs` waits for context close, but context close comes after `saveAs`.

This is a well-documented Playwright behavior:
- [playwright-python#1462](https://github.com/microsoft/playwright-python/issues/1462)
- [playwright#4148](https://github.com/microsoft/playwright/issues/4148)

### 2. Secondary: No timeout on `page.evaluate()` (line ~1700)

```typescript
await bot.page.evaluate(() => {
  if ((window as any).__meetbot_audioRecorder) {
    (window as any).__meetbot_audioRecorder.stop();
  }
});
```

If the page is unresponsive (crashed tab, network issue), this `evaluate()` call has no timeout and can hang indefinitely.

## Fix

**File: `backend/src/bots/google-meet-bot.service.ts`** — `stopBot()` method (line ~1684)

### The correct Playwright video save pattern:

```typescript
// 1. Get video reference BEFORE closing
const video = bot.page.video();

// 2. Close context (finalizes the video file)
await bot.context.close();

// 3. NOW call saveAs (video is finalized, this returns immediately)
if (video) {
  await video.saveAs(screenPath);
}
```

### Full rewrite of `stopBot()` — changes:

1. **Move `video.saveAs()` to AFTER `context.close()`** — get a reference to `page.video()` before closing, then call `saveAs()` after the context is closed (which finalizes the recording).

2. **Add timeout to all `page.evaluate()` calls** — wrap the audio recorder stop in a `Promise.race` with a 3-second timeout so it can't hang.

3. **Add timeout to `context.close()`** — Playwright's `context.close()` can be slow when recording video (issue #4148). Wrap it in a `Promise.race` with a 10-second timeout, falling back to `browser.close()` directly.

4. **Add overall timeout** — Wrap the entire stop sequence in a 15-second hard timeout so it never hangs the HTTP response indefinitely.

### Detailed change:

```typescript
async stopBot(meetingKey: string): Promise<void> {
    const bot = this.activeBots.get(meetingKey);
    if (!bot) return;

    bot.isRunning = false;

    // Clear intervals and timers
    if (bot.flushInterval) clearInterval(bot.flushInterval);
    if (bot.endMonitorInterval) clearInterval(bot.endMonitorInterval);
    if (bot.participantMonitorInterval) clearInterval(bot.participantMonitorInterval);
    if (bot.captionHealthInterval) clearInterval(bot.captionHealthInterval);
    if (bot.hardTimeoutTimer) clearTimeout(bot.hardTimeoutTimer);

    // Stop audio recorder in the page (with timeout)
    if (bot.audioStream) {
      try {
        await Promise.race([
          bot.page.evaluate(() => {
            if ((window as any).__meetbot_audioRecorder) {
              (window as any).__meetbot_audioRecorder.stop();
            }
          }),
          new Promise(resolve => setTimeout(resolve, 3000)),
        ]);
        // Wait briefly for final chunks
        await new Promise(resolve => setTimeout(resolve, 1000));
      } catch {
        // Page might already be closed
      }
      bot.audioStream.end();
      this.logger.log(`Audio recording stream closed for ${meetingKey}`);
    }

    // Get video reference BEFORE closing context (needed for saveAs after close)
    let videoRef: ReturnType<Page['video']> = null;
    if (bot.screenRecordingEnabled && bot.storagePath) {
      try {
        videoRef = bot.page.video();
      } catch {
        // Page might already be closed
      }
    }

    // Try to leave the meeting gracefully
    try {
      const leaveBtn = bot.page.locator(
        '[aria-label*="Leave call" i], [data-tooltip*="Leave call" i]',
      );
      if ((await leaveBtn.count()) > 0) {
        await leaveBtn.first().click({ timeout: 3000 });
        this.logger.debug('Clicked "Leave call" button');
        await bot.page.waitForTimeout(1000);
      } else {
        await bot.page.keyboard.down('Control');
        await bot.page.keyboard.down('Alt');
        await bot.page.keyboard.press('KeyQ');
        await bot.page.keyboard.up('Alt');
        await bot.page.keyboard.up('Control');
        await bot.page.waitForTimeout(1000);
      }
    } catch {
      // Page might already be closed
    }

    // Close browser context (this finalizes the video recording)
    try {
      await Promise.race([
        bot.context.close(),
        new Promise(resolve => setTimeout(resolve, 10_000)),
      ]);
    } catch {
      // Ignore
    }

    // Save screen recording AFTER context close (video is now finalized)
    if (videoRef && bot.storagePath) {
      try {
        const recordingDir = getRecordingDir(bot.storagePath, meetingKey);
        const screenPath = path.join(recordingDir, 'screen.webm');
        await Promise.race([
          videoRef.saveAs(screenPath),
          new Promise((_, reject) =>
            setTimeout(() => reject(new Error('saveAs timeout')), 10_000),
          ),
        ]);
        this.logger.log(`Screen recording saved: ${screenPath}`);
      } catch (err: any) {
        this.logger.warn(`Failed to save screen recording: ${err.message}`);
      }
    }

    // Close the browser
    try {
      await bot.browser.close();
    } catch {
      // Ignore
    }

    this.activeBots.delete(meetingKey);
    this.logger.log(`Bot stopped for meeting ${meetingKey}`);
  }
```

### Key differences from current code:

| Step | Current (broken) | Fixed |
|------|-----------------|-------|
| Audio evaluate | No timeout | 3s timeout via `Promise.race` |
| video.saveAs | Called BEFORE context.close → **deadlock** | Called AFTER context.close → works |
| context.close | No timeout | 10s timeout via `Promise.race` |
| video.saveAs after close | N/A | 10s timeout via `Promise.race` |

**File: `backend/src/bots/google-meet-bot.service.ts`**

Audio recording uses WebRTC track interception + MediaRecorder API:

**Step 1: Inject init script before navigation** (in `runBot()`, before step 1):
```typescript
if (audioRecordingEnabled) {
  await page.addInitScript(() => {
    // Globals for audio capture
    window.__meetbot_audioContext = null;
    window.__meetbot_audioDestination = null;
    window.__meetbot_audioRecorder = null;
    window.__meetbot_audioSources = [];

    // Patch RTCPeerConnection to intercept incoming audio tracks
    const OriginalRTC = window.RTCPeerConnection;
    window.RTCPeerConnection = function(...args) {
      const pc = new OriginalRTC(...args);
      
      pc.addEventListener('track', (event) => {
        if (event.track.kind === 'audio') {
          try {
            if (!window.__meetbot_audioContext) {
              window.__meetbot_audioContext = new AudioContext();
              window.__meetbot_audioDestination = 
                window.__meetbot_audioContext.createMediaStreamDestination();
            }
            
            const stream = new MediaStream([event.track]);
            const source = window.__meetbot_audioContext
              .createMediaStreamSource(stream);
            source.connect(window.__meetbot_audioDestination);
            window.__meetbot_audioSources.push(source);
            
            // Start recorder if not already started
            if (!window.__meetbot_audioRecorder && 
                window.__meetbot_audioDestination) {
              const recorder = new MediaRecorder(
                window.__meetbot_audioDestination.stream,
                { mimeType: 'audio/webm;codecs=opus' }
              );
              recorder.ondataavailable = async (e) => {
                if (e.data.size > 0 && window.__meetbot_onAudioChunk) {
                  const buffer = await e.data.arrayBuffer();
                  const bytes = new Uint8Array(buffer);
                  let binary = '';
                  for (let i = 0; i < bytes.length; i++) {
                    binary += String.fromCharCode(bytes[i]);
                  }
                  window.__meetbot_onAudioChunk(btoa(binary));
                }
              };
              recorder.start(5000); // 5-second chunks
              window.__meetbot_audioRecorder = recorder;
            }
          } catch (err) {
            console.error('Audio capture error:', err);
          }
        }
      });
      
      return pc;
    };
    // Copy static properties and prototype
    Object.setPrototypeOf(window.RTCPeerConnection, OriginalRTC);
    window.RTCPeerConnection.prototype = OriginalRTC.prototype;
  });
}
```

```typescript
async listByUserPaginated(
  userId: string,
  page: number = 1,
  limit: number = 20,
  status?: string,
  search?: string,
): Promise<{ meetings: Meeting[]; meta: PaginationMeta }> {
  const qb = this.meetingsRepository.createQueryBuilder('meeting')
    .where('meeting.userId = :userId', { userId })
    .orderBy('meeting.createdAt', 'DESC');
  
  if (status) {
    qb.andWhere('meeting.status = :status', { status });
  }
  
  if (search) {
    qb.andWhere(
      '(meeting.nativeMeetingId LIKE :search OR meeting.data LIKE :search)',
      { search: `%${search}%` }
    );
  }
  
  const total = await qb.getCount();
  const meetings = await qb
    .skip((page - 1) * limit)
    .take(limit)
    .getMany();
  
  return {
    meetings,
    meta: {
      page,
      limit,
      total,
      totalPages: Math.ceil(total / limit),
    },
  };
}
```

```typescript
function groupMeetingsByDate(meetings: Meeting[]): Map<string, Meeting[]> {
  const groups = new Map<string, Meeting[]>();
  const today = startOfDay(new Date());
  const yesterday = subDays(today, 1);
  
  for (const meeting of meetings) {
    const meetingDate = startOfDay(new Date(meeting.createdAt));
    let label: string;
    
    if (isSameDay(meetingDate, today)) {
      label = 'Today';
    } else if (isSameDay(meetingDate, yesterday)) {
      label = 'Yesterday';
    } else {
      label = format(meetingDate, 'MMMM d, yyyy');
    }
    
    if (!groups.has(label)) groups.set(label, []);
    groups.get(label)!.push(meeting);
  }
  
  return groups;
}
```

```typescript
    if (bot.screenRecordingEnabled) {
      guestContextOptions.recordVideo = {
        dir: getRecordingDir(meetingKey),
        size: { width: 1280, height: 720 },
      };
    }
    
    const newContext = await browser.newContext(guestContextOptions);
    const newPage = await newContext.newPage();
    
    // Re-expose audio chunk function if audio recording
    if (bot.audioRecordingEnabled) {
      await this.setupAudioRecording(newPage, meetingKey);
    }
    
    // Update bot references
    bot.page = newPage;
    bot.context = newContext;
    page = newPage;  // update local var
    context = newContext;
    
    // Re-navigate
    await page.goto(meetingUrl, { waitUntil: 'networkidle', timeout: 60000 });
    await page.waitForTimeout(5000);
    
    // Handle any blocking pages again
    await this.handleBlockingPages(page, meetingUrl);
    
    this.logger.log('Switched to guest mode successfully');
  }
}

// Step 2: Turn off mic/camera (continues as normal)
// Step 4: fillNameIfGuest will fill botName since we're now in guest mode
```

**Key behavior:**
- Only retries if auth was originally provided (`authPath` was truthy)
- Reuses the same browser instance (avoids full relaunch overhead)
- Creates a fresh context without `storageState`
- Updates the `ActiveBot` references so cleanup works correctly
- `fillNameIfGuest()` (step 4) will naturally fill the bot name since we're now in guest mode
- The `botName` from the meeting data is used as the guest name

## Risks and Considerations

1. **Audio recording reliability**: WebRTC track interception depends on patching `RTCPeerConnection` before Google Meet establishes connections. Using `page.addInitScript()` ensures this runs before any page JavaScript. However, if Google Meet changes their WebRTC implementation, this could break.

2. **WebM container validity**: Audio chunks streamed back as raw data may produce a WebM file that lacks proper container headers. To mitigate: the first chunk from `MediaRecorder.start()` includes the WebM header, and subsequent chunks are valid continuations. The `ondataavailable` callback preserves chunk ordering.

3. **Screen recording file size**: At 1280x720, WebM video files can be 50-200 MB per hour. The STORAGE_PATH should have adequate disk space.

4. **SQLite pagination with LIKE**: Search queries using `LIKE` on the JSON `data` column will do a full table scan. For the expected dataset size (hundreds to low thousands of meetings per user), this is acceptable with SQLite. If performance becomes an issue, consider adding indexed columns for searchable fields.

5. **Guest fallback timing**: Retrying as guest adds ~10-15 seconds to the join flow (close context + create new context + re-navigate). This is acceptable since the alternative is complete failure.
